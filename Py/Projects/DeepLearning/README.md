
# Deep Learning 课程实践总结

本仓库包含深度学习基础与序列建模的练习代码，旨在通过代码实现理解从基础感知机到现代 Transformer 架构的演进逻辑。

## 📁 目录结构

* `Class_1.py`: 前馈神经网络 (MLP)、二分类任务训练流程与正则化技术。
* `Class_2.py`: 序列建模进阶，涵盖 RNN 基础、自注意力 (Self-Attention) 及 Transformer Block 实现。

---

## 🟢 第 1 课：前馈神经网络与训练正则化

本节重点在于跑通**监督学习的完整工程链路**，理解如何构建稳定且具备泛化能力的模型。

### 1.1 核心目标
* 搭建基础二分类神经网络。
* 掌握训练集/验证集划分、损失函数选择及优化器配置。
* **正则化实战**：对抗过拟合，提升模型泛化性能。
* **工程化能力**：模型保存 (Checkpoint)、早停 (Early Stopping) 及训练可视化。

### 1.2 关键技术栈
| 概念 | 说明 |
| :--- | :--- |
| **MLP** | 多层感知机，深度学习的基础。 |
| **BCEWithLogitsLoss** | 二分类首选，集成 Sigmoid 与交叉熵，数值更稳定。 |
| **Dropout** | 训练时随机失活，强制模型学习鲁棒特征。 |
| **AdamW** | 改进型 Adam 优化器，修复了权重衰减 (Weight Decay) 的耦合问题。 |
| **Early Stopping** | 监控验证集损耗，防止过拟合的最后一道防线。 |



### 1.3 核心逻辑流
```text
数据准备 -> 数据增强 -> 模型定义 -> 划分数据集 -> 训练循环 (反向传播) -> 验证评估 -> Checkpoint 保存 -> 可视化
```

> **🔑 必记结论**：模型不仅要“学会”训练集，更要学会“泛化”。正则化的本质是约束模型的复杂度。

## 🔵 第 2 课：序列建模，从 RNN 到 Transformer

本节探讨当数据具备**时序性**和**上下文依赖**时，架构该如何演进。

### 2.1 循环神经网络 (RNN)
RNN 通过维护一个随时间步更新的**隐藏状态 ($h_t$)** 来处理序列：
$$h_t = f(x_t, h_{t-1})$$

* **优势**：共享参数、处理变长序列、天然符合时序直觉。
* **局限**：**长期依赖丢失**（梯度消失/爆炸）以及**串行计算瓶颈**（无法完全发挥 GPU 并行算力）。



### 2.2 注意力机制 (Attention)
为了解决 RNN 路径过长的问题，Attention 允许模型“直接观察”全局：
1.  **Q (Query)**：当前词在找什么？
2.  **K (Key)**：其他词能提供什么？
3.  **V (Value)**：其他词实际的内容。

**核心公式：**
$$Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

### 2.3 Transformer 架构
Transformer 彻底抛弃了循环结构，转而依靠**位置编码 (Positional Encoding)** 和 **自注意力 (Self-Attention)**。

**核心模块：**
* **Input Embedding & Positional Encoding**：注入语义与位置信息。
* **Multi-Head Attention**：多视角并行捕捉上下文。
* **Feed Forward Network (FFN)**：逐位置的非线性特征提取。
* **Residual + LayerNorm**：确保深层网络训练的稳定性。

## 🤝 两门课程的演进逻辑

从工程基础到架构创新的演进路径：

1.  **第 1 课（工程基石）**：解决“怎么训练、怎么优化、怎么保存”的问题。
2.  **第 2 课（架构演进）**：解决“如何建模复杂关联、如何实现并行化”的问题。

### 总结对照表
| 特性 | MLP / 前馈网络 | RNN / 循环网络 | Transformer |
| :--- | :--- | :--- | :--- |
| **输入关系** | 独立同分布 | 线性时序依赖 | 全局语义关联 |
| **计算效率** | 高 (并行) | 低 (串行) | 极高 (并行) |
| **长程记忆** | 无 | 弱 (受限于梯度消失) | 强 (一步到位) |
